The Ingest LDD schema is defined in the PDS4 Information Model (IM) and is used to define local data dictionaries. Once a local data dictionary is defined it can be ingested into the IM and the PDS4 Data Dictionary Data Base (DDDB). A local data dictionary at the discipline level is required to be ingested. A dictionary at the mission level can optionally be ingested.

The PDS4 Information Model (IM) is the fundamental reference for PDS4 structure; its requirements can be validated automatically using eXtensible Markup Language (XML) schemas.

The PDS4 Data Dictionary Data Base (DDDB) is the fundamental reference for definitions of classes and attributes.
